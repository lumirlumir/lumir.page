리액트에서 클래스 문법 대신 함수 문법을 사용하게 된 이유는 여러 가지가 있습니다. 주요 이유는 다음과 같습니다:

1. **단순성과 가독성**:
   함수형 컴포넌트는 일반적으로 클래스형 컴포넌트보다 더 단순하고 직관적입니다. 코드가 더 간결해지고 가독성이 높아지며, 함수형 프로그래밍 패러다임에 익숙한 개발자들에게 더 익숙한 문법입니다.

2. **React Hooks의 도입**:
   React 16.8에서 도입된 Hooks는 함수형 컴포넌트에서도 상태 관리와 사이드 이펙트를 다룰 수 있게 해줍니다. Hooks를 사용하면 클래스 컴포넌트에서만 가능했던 기능들을 함수형 컴포넌트에서도 쉽게 구현할 수 있습니다. 대표적인 Hooks로는 `useState`, `useEffect`, `useContext` 등이 있습니다.

3. **this 키워드의 혼란 방지**:
   클래스형 컴포넌트에서는 `this` 키워드의 사용이 필수적이며, 이는 종종 혼란을 초래합니다. 특히, `this`의 바인딩 문제는 자바스크립트의 초보자에게 큰 어려움이 될 수 있습니다. 함수형 컴포넌트는 이러한 문제를 완전히 피할 수 있습니다.

4. **컴포넌트 간 공유 로직의 용이성**:
   Hooks는 여러 컴포넌트 간에 로직을 쉽게 공유할 수 있는 방법을 제공합니다. 예를 들어, 사용자 정의 Hook을 만들어 여러 컴포넌트에서 동일한 상태 관리 로직을 재사용할 수 있습니다. 클래스 컴포넌트에서는 이를 위해 HOC(High Order Components)나 Render Props 패턴을 사용해야 했는데, 이들은 비교적 복잡할 수 있습니다.

5. **성능 향상**:
   함수형 컴포넌트는 메모리 사용 측면에서 더 효율적일 수 있습니다. 클래스형 컴포넌트는 인스턴스를 생성하고 메모리를 차지하지만, 함수형 컴포넌트는 그저 함수 호출이기 때문에 상대적으로 메모리 사용량이 적습니다.

6. **미래 지향적 기능들**:
   React는 앞으로 더 많은 기능을 함수형 컴포넌트와 Hooks 중심으로 제공할 계획입니다. 이는 React 팀이 함수형 컴포넌트를 중심으로 한 생태계를 더 발전시키고 있다는 의미입니다.

이러한 이유들로 인해 React에서는 클래스 컴포넌트 대신 함수형 컴포넌트와 Hooks를 사용하는 것이 권장되고 있습니다. 이를 통해 더 단순하고 유지보수하기 쉬운 코드를 작성할 수 있습니다.
