When JavaScript(***JavaScript***) first appeared, it was merely a scripting language responsible for tasks such as data validation on the web browser (client) side.

However, today it interacts with almost every part of browser windows and content, and it even handles closures, anonymous (lambda) functions, and metaprogramming. It has become a full programming language capable of complex calculations and diverse interactions.

Later, the appearance of ***jQuery*** made it easy to handle the ***DOM***, and the appearance of ***Node.js*** made server development with ***JavaScript*** possible.

## ***JavaScript*** vs ***ECMAScript***

### ***ECMAScript***

ECMA is short for European Computer Manufacturer's Association, an international standards organization for information and communication systems. (It has now changed its name to ECMA International.) ***ECMAScript*** refers to the standard for scripting languages such as ***JavaScript***. ***JavaScript*** is known as the most popular language based on ***ECMAScript***.

### ***JavaScript***

The original purpose of ***JavaScript*** was to implement dynamic elements in the Netscape web browser. After that, many other web browsers also began to include ***JavaScript***, and a standard specification became necessary so that ***JavaScript*** could work consistently across different browsers. This is why ECMA created the scripting standard. ***JavaScript*** is a general-purpose scripting language that follows the ***ECMAScript*** specification, and this kind of standard scripting language is called ***ECMAScript***.

In other words, you can think of ***ECMAScript*** as a kind of manual that explains how a scripting language should be made, and ***JavaScript*** is a language created based on the ***ECMAScript*** specification.
